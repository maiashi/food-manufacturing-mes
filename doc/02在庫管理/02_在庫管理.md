# 食品製造業MES - 在庫管理要件

## 1. 目的
食品製造の生産現場で必要となる在庫管理機能の要件を定義する。
FIFO（先入先出）の厳格な実施、ロットの分割処理、保管場所の追跡を含む。

---

## 2. 基本設計方針

### 2.1 ロット管理の基本
- 各物品は「ロット」単位で在庫を管理する
- ロットには固有のLot ID（一意）が振られる
- 原材料・半成品・完成品全ての在庫に適用
- 有効期限（賞味/消費期限日付）を必ず持つ

### 2.2 FIFO管理の基本
- 出荷・仕入先への投入時は、最も古いロット（製造日/受入日順）から優先する
- システムが自動的にFIFO順の発注先を提案
- FIFO違反を防ぐためのチェックと警告

### 2.3 ロット分割の基本
- 原材料の在庫から一部を使用して製品を作る際、元ロットは分割管理される
- 分割後の各ロットは独立した有効期限・保管場所を持つ
- 分割元のロット情報との追跡関係を維持（親-子関係）

## 4. FIFO管理の詳細仕様

### 4.1 FIFOルール
```
FIFO原則: 同じ物品で同じ保管条件を持つロット群について、
          有効期限が最も早いもの → または 製造日が最も古いもの
          の順に優先して引き出す。
```

### 4.2 優先順位
1. **同一有効期限内**: 製造日/受入日の早い順
2. **異なる有効期限**: 有効期限の早い順（これを最優先）
3. **例外処理**: アレルギー管理上、特定ロットを優先/回避する場合は特別指示

### 4.3 FIFOチェック
- 仕入先投入時: システムがFIFO違反を検知し警告
- 出荷時: システムがFIFO違反を検知し警告（理由は必須入力）
- 記録: 全ての例外は監査証跡に記録

## 4. カテゴリ別賞味期限・FIFO要件

本工場で扱う4つのカテゴリにより、賞味期限の長さやFIFOの厳格度が大きく異なる。
システムはカテゴリ毎に異なるルールで在庫管理を行う必要がある。

### 4.1 賞味期限マスタ（カテゴリ別デフォルト）

| カテゴリ | SKUプレフィックス | デフォルト賞味日数 | 保管温度 | FIFO優先度 | 警告閾値 |
|---------|-----------------|-------------------|---------|-----------|---------|
| レトルト | RC-XXX | 製造日から18ヶ月(540日) | 常温(15〜25℃) | ★☆☆ 低 | 賞味期限6ヶ月前 |
| チルド総菜 | TC-XXX | 製造日から7日 | 冷蔵(0〜5℃) | ★★★ 高 | 残り3日で警告、残り1日で出荷停止 |
| デリカ | DL-XXX | 製造日から3日 | 冷蔵(0〜4℃) | ★★★★ 最高 | 残り1日で警告、当日限りで出荷停止 |
| おせち（レトルト） | RC-OSECHI | 商習慣:12月31日まで | 常温 | ★★☆ 中 | 12月25日で出荷停止 |
| おせち（チルド構成品）| TC-OSECHI | 製造日から7日（但し12月31日まで） | 冷蔵(0〜5℃) | ★★★ 高 | 残り3日で警告 |

### 4.2 カテゴリ別FIFO実行ルール

#### レトルト食品
- FIFOの優先度は比較的低い。賞味期限が長いため、若干の遅延は許容範囲内
- ただし**同一原材料ロットによる多製品共用**の場合は注意（例: 鶏肉を複数のレトルト製品で共用）
- 季節もの(oisechi)は通常品より優先して出荷

#### チルド総菜
- **厳格なFIFO必須**。賞味7日なので、1日出荷が遅れると約14%の寿命消耗
- システムによる自動在庫割当て：受注入力時に自動的に最も古いLotを確定
- 手動変更は管理者権限のみ。変更時は理由入力（全量廃棄覚悟で変更等）

#### デリカ
- **最優先FIFO**。賞味3日で半分以上は製造当日に消費される
- 製造→倉庫入库→出荷のサイクルを24時間以内に完了するようシステムが監視
- FIFO違反検知時は即時アラート（画面・メール・チャット通知）

#### おせち料理
- 通常期間(1月〜11月): 基本生産なし。構成品の原材料管理のみ
- 生産期間(12月1日〜20日): 大量生産のため、Lot ID管理が複雑になる
- 出荷期間(12月21日〜30日): 「12月31日まで」商習慣に従う。日付ベースで優先

### 4.3 ロット分割時のFIFO影響

デリカ・チルドでは原材料のロット分割が頻発する。例えば:
```
LOT-R-0501 (鶏肉 20kg) ← 受入日: 5/10, 賞味: 5/13（残り3日!）

バッチA(RC-001)で8kg使用 → LOT-R-0501残12kg
バッチB(TC-001)で6kg使用 → LOT-R-0501残6kg  
バッチC(DL-001)で4kg使用 → LOT-R-0501残2kg
  ↑ ここまで全てFIFO順。LOT-R-0501の賞味3日以内に全て使い切れない場合、廃棄リスク

新ロットLOT-R-0510 (鶏肉 30kg) ← 受入日: 5/12, 賞味: 5/15（残り3日）
バッチD(DL-001)で3kg使用 → LOT-R-0501(残2kg)+LOT-R-0510(1kg)の分割使用
```

システムは以下を自動判断:
1. 各原材料の在庫Lotを賞味期限昇順でソート
2. バッチの必要量に対してFIFO順に引き当て
3. 1つのLotで足りなければ次の古いLotから継続（分割）
4. 全て引き当て完了後に警告チェック（廃棄リスク・賞味不足等）
