# 食品製造業MES - 保管場所管理要件

## 1. 目的
食品製造工場の在庫保管場所を階層的に管理し、温度帯・湿度条件等の保存環境と連動させて、
各ロットの正確な保管位置を追跡する機能の要件を定義する。

---

## 2. 基本設計方針

### 2.1 保管場所の階層構造
保管場所は3層以上の階層で管理される:

```
倉庫 (Warehouse)
 └── 棟/ゾーン (Zone)
     └── 通路/区域 (Aisle/Area)
         └── 棚/ラック (Shelf/Rack)
             └── 位置 (Position/Bin)
```

### 2.2 温度帯管理
食品工場では保管温度帯が重要な管理項目である:
- 常温倉庫（15〜25℃）
- 冷蔵倉庫（0〜10℃）
- 冷凍倉庫（-18℃以下）
- 特冷倉庫（-30℃以下）

各保管場所は温度帯を属性として持ち、物品の保存条件と自動マッチする。

## 3. 機能要件

### 3.1 倉庫マスタ管理

| ID | 要求項目 | 詳細 |
|----|---------|------|
| WH-001 | 倉庫コード | 固有のWarehouse ID（例: WH-01, WH-02） |
| WH-002 | 倉庫名 | 分かりやすい名称 |
| WH-003 | 温度帯 | 常温/冷蔵/冷凍/特冷 |
| WH-004 | 容量制限 | 総容積・総重量の設定と残量管理 |
| WH-005 | 倉庫種別 | 原材料倉庫/半成品倉庫/完成品倉庫/返品倉庫等 |

### 3.2 ゾーンマスタ管理

| ID | 要求項目 | 詳細 |
|----|---------|------|
| ZN-001 | ゾーンコード | 倉庫内ゾーンID（例: WH-01-ZA） |
| ZN-002 | ゾーン名 | ロードドック/仕分けゾーン等 |
| ZN-003 | 温度帯継承/上書き | 親倉庫からの継承または個別設定 |
| ZN-004 | ゾーン種別 | 原料Zone/調合Zone/加熱Zone/冷却Zone等 |

### 3.3 棚・ラックマスタ管理

| ID | 要求項目 | 詳細 |
|----|---------|------|
| SH-001 | 棚コード | 固有のShelf ID（例: A-01-03 = A通路, 01段目, 03番） |
| SH-002 | 親ゾーン参照 | 所属するゾーンを指定 |
| SH-003 | 積載上限 | 重量(kg)/容積(L)の上限設定 |
| SH-004 | 使用可否 | 使用中/点検中/使用不可 |

### 3.4 保管場所と物品の対応管理

| ID | 要求項目 | 詳細 |
|----|---------|------|
| LOC-001 | 現在位置検索 | ロットIDから現在の保管場所を即座に取得 |
| LOC-002 | 逆引き検索 | 保管場所IDからそこに保管中の全ロットを取得 |
| LOC-003 | 移動履歴 | 各ロットの保管場所移動履歴を追跡 |
| LOC-004 | 誤配置検知 | 冷蔵品が常温棚に置かれた等、条件違反を検知 |

### 3.5 保管場所移動（Transfer）

| ID | 要求項目 | 詳細 |
|----|---------|------|
| TRF-001 | 移動伝票 | ロット・移動元・移動先・移動理由を記録 |
| TRF-002 | 移動チェック | 移動先の温度帯が物品の保存条件と適合するか自動検証 |
| TRF-003 | 移動承認 | 管理者による承認フロー（任意設定） |
| TRF-004 | RFID/バーコード対応 | 物理移動時の記録を簡素化 |

### 3.6 保管環境モニタリング連携

| ID | 要求項目 | 詳細 |
|----|---------|------|
| MON-001 | 温度ログ取得 | IoT温度センサーとの連携でリアルタイム温度を取得 |
| MON-002 | 温度逸脱警告 | 設定範囲外の温度を検知時に通知 |
| MON-003 | 温度履歴レポート | 保管期間中の温度変化をグラフ表示 |

### 3.7 保管場所利用率レポート

| ID | 要求項目 | 詳細 |
|----|---------|------|
| RPT-001 | 倉庫別占有率 | 各倉庫の容量使用率（%）をリアルタイム表示 |
| RPT-002 | 棚別充填率 | 各棚の使用状況（埋まっている/空いている）を可視化 |
| RPT-003 | 温度帯別在庫分布 | 冷蔵/冷凍/常温の在庫量バランスを把握 |

## 4. 保管場所コード体系（推奨）

```
形式: {通路}{段目}{位置}
例: A-01-03

意味:
  A = 第A通路（またはAゾーン）
  01 = 1段目（棚の高さ位置）
  03 = その段の3番目の保管ボックス/場所
```

拡張: 複数階層が必要な場合は
```
{倉庫}{棟}{通路}{段目}{位置}
例: WH01-B-A-01-03
```

## 5. 温度帯適合チェックの仕様

```python
# 保管場所移動時の適合チェックロジック（参考）

def check_temperature_compatibility(product, destination_location):
    required_temp = product.storage_condition  # e.g., "frozen"
    location_temp = destination_location.temperature_zone  # e.g., "cold"
    
    compatibility_matrix = {
        ("常温", ["常温"]),
        ("冷蔵", ["常温", "冷蔵"]),
        ("冷凍", ["常温", "冷蔵", "冷凍"]),
        ("特冷", ["常温", "冷蔵", "冷凍", "特冷"]),
        ("温度帯不限定", ["常温", "冷蔵", "冷凍", "特冷"]),  # 例: 乾燥品
    }
    
    if required_temp not in compatibility_matrix.get(location_temp, []):
        raise TemperatureViolationError(
            f"物品({product.name})の保存条件({required_temp})と、"
            f"保管場所({destination_location.code})の温度帯({location_temp})が適合しません。"
        )
    return True
```

## 6. 保管場所移動フロー

```
1. 移動伝票作成（Lot ID, 移動元, 移動先, 理由）
2. システムチェック:
   a) 温度帯適合チェック
   b) 積載上限チェック
   c) 他物品との混载制限チェック（アレルゲン等）
3. チェック通過 → 移動実行
4. 保管場所変更を即時反映（リアルタイム更新）
5. 監査証跡に記録（Who/When/Why/LotID）
```

## 6. カテゴリ別保管場所要件（温度区分・混在管理）

### 6.1 温度区分ゾーン定義

| ゾーンID | 名称 | 温度範囲 | 対象カテゴリ | 主要SKUプレフィックス |
|---------|------|---------|-------------|---------------------|
| ZONE-COLD | 冷凍倉庫 | -25〜-18℃ | 原材料（肉・魚介） | RM-FREEZE-* |
| ZONE-CHILL-1 | 冷蔵A庫 | 0〜3℃ | デリカ原材料・完成品 | RM-DELI-*, DL-* |
| ZONE-CHILL-2 | 冷蔵B庫 | 0〜5℃ | チルド総菜完成品 | TC-* |
| ZONE-CHILL-3 | 冷蔵C庫（おせち用）| 0〜5℃ | おせちチルド構成品 | TC-OSECHI-* |
| ZONE-ROOM | 常温倉庫 | 15〜25℃、湿度60%以下 | レトルト完成品 | RC-*, BOX-* |
| ZONE-PREP | 前処理室 | 10〜15℃ | 原材料前処理中 | WIP-* |

### 6.2 カテゴリ別の保管場所フロー

#### レトルト食品
```
殺菌・冷却後 → ラベリング → 常温倉庫(ZONE-ROOM)入库
    ↓（出荷指示）→ ZONE-ROOMからピッキング → 出荷
```
- 常温倉庫は湿度管理も重要。レトルト袋のラベル劣化防止のため湿度60%以下

#### チルド総菜
```
冷却後 → ラベリング → 冷蔵B庫(ZONE-CHILL-2)入库
    ↓（出荷指示）→ FIFO順にピッキング → 保冷ボックス搭載トラックへ
```
- 冷蔵B庫は連続温度モニター。5℃超え30分継続時は即時アラート
- 出库時、トラック内の保冷ボックス温度確認を必須

#### デリカ
```
充填・密封後 → ラベリング + アイスパック同梱 → 冷蔵A庫(ZONE-CHILL-1)入库
    ↓（出荷指示）→ FIFO最優先でピッキング → 保冷ボックス(アイスパック追加)搭載トラックへ
```
- 冷蔵A庫は目標3℃以下。デリカ専用ゾーンを設け、他製品との温度混在を防ぐ
- アイスパックの在庫管理も必要（再利用可能タイプか使い切りか）

#### おせち料理
```
各構成品の完成 → カテゴリ別倉庫に一時保管(ZONE-CHILL-2 or ZONE-ROOM)
    ↓（おせち組み立て指示: 12月中）→ 前処理室(ZONE-PREP)で組み立て
    ↓（組み立て後）→ 温度区分に応じた保管
       - レトルト構成品混在: ZONE-ROOM + ZONE-CHILL-3 両方に区分配置
       - 全チルド構成品: ZONE-CHILL-3 に一括保管
    ↓（12月21日以降出荷）→ ピッキング → ギフト包装 → 保冷ボックス搭載
```
- おせち期間は通常倉庫を一部おせち専用ゾーンに切り替え
- レトルトとチルド構成品が同じ重箱に入るため、保管場所の温度区分表示が重要

### 6.3 保管場所割り当てルール

1. **SKU毎に保管可能ゾーンを定義**: システムで製品マスターに許可ゾーンを紐付け
2. **入库時に自動判別**: SKUからゾーンを自動選択。手動変更は管理者のみ
3. **混在禁止ゾーン**: デリカ(ZONE-CHILL-1)にはチルド総菜(TC-*)やレトルト(RC-*)の入库不可
4. **おせち期間(12月)の特殊ルール**: 
   - 通常はRC-*とTC-*を別ゾーンに配置
   - おせち専用エリア(ZONE-OSECHI)では温度区分ごとに物理的に仕切り
5. **FIFOによる保管位置案内**: システムが「最も賞味期限が近いLot」の保管位置をピッキング先に通知
