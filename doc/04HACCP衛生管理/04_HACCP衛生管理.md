# 食品製造業MES - HACCP衛生管理要件

## 1. 目的
HACCP（危害要因分析重要管理点）方式に基づく衛生管理をシステム上で実施・記録・検証するための要件を定義する。
2021年6月の食品衛生法改正で、全ての食品事業者のHACCP導入が義務化されている。

---

## 2. HACCP方式の種類（システム対応）

### 2.1 3つのアプローチ

| タイプ | 名称 | 対象 | システム対応 |
|--------|------|------|-------------|
| ① | 自主的・合理的に構築される衛生管理 | 中小規模事業者等 | プラン登録、記録入力 |
| ② | 適正化計画に基づく構築 | 食品供給連鎖(FSC)参加者 | FSC規格準拠のテンプレート |
| ③ | 認定認証機関等による確認・助言 | JFS/FSSC22000認証取得希望 | 認証対応レポート出力 |

- [REQ-HACCP-001] 自社工場のHACCP方式タイプを選択可能にする
- [REQ-HACCP-002] タイプに応じたテンプレート・チェックリストを提供

## 3. プリ・レquisites（実施衛生プログラム）管理

プリ・レquisitesはHACCPの基礎となる衛生管理体制。システムで以下のプログラムを管理する:

### 3.1 施設・設備の保守管理

| ID | 要求項目 | 詳細 |
|----|---------|------|
| PR-001 | 清掃スケジュール | 場所・頻度・担当者・方法を登録 |
| PR-002 | 清掃実施記録 | 毎日/毎回の清掃結果を入力・記録 |
| PR-003 | 消毒記録 | 消毒剤の種類・濃度・実施日時を記録 |
| PR-004 | 設備点検記録 | 機械の定期点検日をスケジュール・実績を記録 |
| PR-005 | 害虫防除記録 | 殺虫・駆除の実施履歴（日付・業者・結果） |

### 3.2 従業員の衛生管理

| ID | 要求項目 | 詳細 |
|----|---------|------|
| PR-006 | 健康診断記録 | 年次健康診断の結果を登録 |
| PR-007 | 感染症届出 | インフルエンザ等、食品製造に支障が出る疾病の届出 |
| PR-008 | 手洗い・消毒実施状況 | 入場時の手洗い実施記録 |
| PR-009 | 服装管理 | 作業服の清潔度チェック記録 |

### 3.3 原材料の安全管理

| ID | 要求項目 | 詳細 |
|----|---------|------|
| PR-010 | 受入検査記録 | 到着時の外観・温度・規格適合性を記録 |
| PR-011 | サプライヤー審査 | 供給者の品質・衛生実績を定期評価 |
| PR-012 | COA（Certificate of Analysis）管理 | 原材料の分析成績書の保存・参照 |

## 4. HACCP 7原則の実施と記録管理

### 原則1: 危害要因分析（Hazard Analysis）

#### 危害要因の種類
- **生物的**: リステリア、サルモネラ、大腸菌O157等
- **化学的**: 農薬残留、重金属、食品添加物の過剰使用、洗浄剤残留等
- **物理的**: ガラス片、金属片、プラスチック片、石等

| ID | 要求項目 | 詳細 |
|----|---------|------|
| HA-001 | 危害要因登録 | 各工程における生物/化学/物理的危害要因を登録 |
| HA-002 | 重大性評価 | 各危害の発生可能性・重大性を1〜5段階で評価 |
| HA-003 | 許容可能性判定 | 評価結果に基づきCCP指定可否を判断 |
| HA-004 | 分析レポート | HACCP計画書の一部としてエクスポート可能 |

**危害要因登録フォーム例:**
```
工程: [加熱処理]
危害要因: リステリア属菌の残留（生物的）
発生可能性: [低/中/高] (1-5)
重大性: [軽微/中等/重大] (1-5)
是正措置: 72℃以上30分間の加熱
CCP指定: [是/否]
```

### 原則2: 重要管理点（CCP）の特定

| ID | 要求項目 | 詳細 |
|----|---------|------|
| CCP-001 | CCPマップ | プロセスフロー図上にCCPをマッピング表示 |
| CCP-002 | CCP決定ツリー | デシジョンツリーを用いたCCP判定の記録 |
| CCP-003 | CCPリスト | 工場の全CCPを一覧管理 |

**CCP登録情報:**
- CCP ID, 工程名, CCP番号（例: CCP-1, CCP-2）
- 制御対象危害要因
- クリティカルリミット（限界値）
- 監視方法・頻度
- 是正措置
- 検証方法

### 原則3: クリティカルリミット（限界値）の設定

| ID | 要求項目 | 詳細 |
|----|---------|------|
| CL-001 | 限界値登録 | 温度・時間・pH・Aw・塩濃度等のCCP限界値を設定 |
| CL-002 | 限界値タイプ | 数値範囲（例: 72〜85℃）、最小値/最大値 |
| CL-003 | 複合リミット | 温度×時間（例: 72℃以上 かつ 30分以上）の指定 |

### 原則4: 監視システムの設定

| ID | 要求項目 | 詳細 |
|----|---------|------|
| MS-001 | CCP監視記録入力 | 製造時に各CCPの実測値を入力（画面/モバイル/自動計器連携） |
| MS-002 | 自動計器連携 | 温度センサー、pHメーター等のデータを自動取得 |
| MS-003 | 監視頻度管理 | CCPごとの監視間隔（例: 30分ごと、毎バッチ）を管理 |
| MS-004 | 担当者設定 | 各CCPの監視担当者を割り当て |

### 原則5: クリティカルリミット逸脱時の是正措置

| ID | 要求項目 | 詳細 |
|----|---------|------|
| CA-001 | 逸脱検知 | CCP実測値がクリティカルリミットから外れた時に自動検知・警告 |
| CA-002 | 是正手順の提示 | 逸脱内容に応じて、事前に登録された是正手順を表示 |
| CA-003 | 是正記録 | 実施した是正処置を記録（対象ロットの数、処分方法等） |
| CA-004 | 影響範囲評価 | 逸脱の影響を受けた他のロットを自動で特定 |

**逸脱対応フロー:**
```
CCP逸脱検知 → 即時警告（画面+通知）→ 是正手順表示 → 
是正処置実行 → 結果入力 → 監査証跡に記録 → 
必要に応じて影響範囲評価・回収判定
```

### 原則6: 検証手順の確立

| ID | 要求項目 | 詳細 |
|----|---------|------|
| VA-001 | CCP検証計画 | 月次/四半期のCCP管理有効性検証スケジュール |
| VA-002 | 機器校正記録 | 計測器の校正日程・結果を記録（温度計、pHメーター等） |
| VA-003 | HACCP計画見直し | 製品変更・工程変更時のHACCP計画の見直し記録 |
| VA-004 | 内部監査管理 | 監査計画→実施→報告→是正措置の全フロー管理 |
| VA-005 | 統計解析 | CCP監視データの傾向分析（管理図等） |

### 原則7: 記録文書の維持

| ID | 要求項目 | 詳細 |
|----|---------|------|
| RC-001 | HACCP記録一元化 | プリ・レquisites、CCP監視、是正措置等の全記録を一元管理 |
| RC-002 | 改竄防止 | 記録入力後の修正は追記のみ（元の値保持）、電子署名 |
| RC-003 | 自動タイムスタンプ | 全記録に日付・時刻の自動付与 |
| RC-004 | エクスポート対応 | PDF/CSV形式での監査用エクスポート |
| RC-005 | 検索機能 | 期間・工程・CCP番号等で記録を検索可能 |

## 5. カテゴリ別CCP要件（製品カテゴリ固有の重要管理点）

本工場で取り扱う4つの製品カテゴリに応じた、必須のCCPを定義する。

### 5.1 レトルト食品

レトルト食品は加熱殺菌工程が唯一かつ最優先のCCP。
 botulism（ボツリヌス菌）等の芽胞性細菌を不活化することが生死に関わる。

| CCP番号 | 工程 | 制御対象危害要因 | クリティカルリミット | 監視方法 | 監視頻度 |
|---------|------|-----------------|---------------------|---------|---------|
| CCP-1 | 殺菌（レトルト加熱） | ボツリヌス菌等芽胞性細菌（生物） | 中心温度≥121℃、かつF値≥3.0以上 | 連続温度記録装置（自動） | 全バッチ連続監視 |
| CCP-2 | パック密閉検査 | パック破綻・漏洩による再汚染（物理） | 加圧漏水テストで漏れなし | バッチ毎サンプル検査（1%以上） | バッチ毎 |
| CCP-3 | 冷却工程 | リステリア等の再生長抑制（生物） | 60℃→10℃まで24時間以内、中心温度≤50℃を2時間以内 | 温度プロファイル記録 | バッチ毎 |

**レトルト特有の注意点:**
- F値（殺菌強度）は温度と時間の積算値。温度計測器の校正が必須
- 充填物（油脂含有率高いもの）は熱伝導が遅く、中心温度到達に時間がかかる。製品別F値計算が必要
- 季節もの（おせち料理のレトルト版）は多品目少量生産のため、殺菌条件設定の変更管理が重要

### 5.2 チルド総菜（冷蔵調理食品）

チルド総菜は加熱後速やかに冷却し、冷蔵で保管する。
冷却速度と保存温度管理が生命線。食中毒菌の増殖防止が最優先。

| CCP番号 | 工程 | 制御対象危害要因 | クリティカルリミット | 監視方法 | 監視頻度 |
|---------|------|-----------------|---------------------|---------|---------|
| CCP-1 | 加熱調理 | 食中毒菌の不活化（生物） | 中心温度≥75℃、1分以上保持 | 中心温度計測 | バッチ毎 |
| CCP-2 | 冷却工程 | 食中毒菌の増殖抑制（生物） | 60℃→10℃まで**4時間以内**（JFS規格） | 温度プロファイル記録 | バッチ毎 |
| CCP-3 | 冷藏保管 | リステリア等の増殖（生物） | 保管温度≤5℃ | 連続モニターまたは2時間毎測定 | 連続/2時間毎 |
| CCP-4 | パック密封 | 再汚染（生物・物理） | シール強度規格適合 | サンプリング検査 | バッチ毎 |

**チルド総菜特有の注意点:**
- **冷却速度が極めて重要**: チルドは賞味3〜7日と短く、冷却不良は一生涯の信頼喪失につながる
- 容器形状（金属製vs樹脂製）で冷却速度が変わる。製品別冷却条件設定が必要
- 冷蔵保管中の温度管理をIoTセンサーで連続監視。逸脱時は即時アラート

### 5.3 デリカ（デラックス惣菜・サラダ等）

デリカは加熱工程を持たないものも多く、原材料の安全性と低温管理が全て。
賞味1〜3日という超短期間の為、FIFO管理の厳格さが必須。

| CCP番号 | 工程 | 制御対象危害要因 | クリティカルリミット | 監視方法 | 監視頻度 |
|---------|------|-----------------|---------------------|---------|---------|
| CCP-1 | 原材料受入（生野菜・肉類） | 食中毒菌の混入（生物） | 温度≤10℃、外観異常なし | 受入検査 | 全回 |
| CCP-2 | 洗浄・切り分け | 残留農薬・異物混入（化学・物理） | 水道水残存塩素0.1〜0.3ppm | 塩素濃度試験紙 | 1時間毎 |
| CCP-3 | 充填・密封 | 再汚染（生物） | 空気清浄度の管理（必要に応じてクリーンルーム） | 微生物検査 | 週次 |
| CCP-4 | 冷藏保管・輸送 | リステリア等の増殖（生物） | ≤5℃ | 連続モニター | 連続/2時間毎 |

**デリカ特有の注意点:**
- **加熱杀菌工程がない**: 原材料段階での安全性確保が最重要。COA（分析成績書）の確認を必須化
- リステリアは0℃でもゆっくり増殖する。冷蔵温度は3〜4℃以下を目標
- アレルギー混入リスクが高い（複数食材の混合調理）。製造ライン切り替え時の洗浄記録を厳格に管理

### 5.4 おせち料理（季節もの・構成食）

おせちは複数のレトルト・チルド食品を組み合わせた構成食。
年末年始に生産が集中する為、**大量生産体制の短期稼働**に対応する必要がある。

| CCP番号 | 工程 | 制御対象危害要因 | クリティカルリミット | 監視方法 | 監視頻度 |
|---------|------|-----------------|---------------------|---------|---------|
| CCP-1〜N | 各構成食品の加工工程 | カテゴリ別CCPに準拠 | レトルト: CCP-1に準拠 チルド: CCP-1,2に準拠 | カテゴリ別に設定 | バッチ毎 |
| CCP-X | 組み立て・包装 | 異物混入・アレルギー混入（物理・化学） | アレルギー表示の正確性、異物混入なし | 外観検査 | 全数 |
| CCP-Y | 完成品保管 | 温度管理（製品によって異なる） | レトルト:常温 チルド:≤5℃ | 倉庫内温度モニター | 連続 |

**おせち特有の注意点:**
- **季節集中生産**: 12月下旬に年間売上の7割以上を売り切る。生産計画・在庫計画が命
- 構成品目が多い（黒豆・田作り・数の子等10〜20点）。BOM管理が複雑になる
- ギフト包装仕様。箱単位でのLot ID発行が必要（箱内の各品目が別のLotの場合あり）
- 賞味期限: 基本的に製造日の1週間前〜月末（商習慣）。商品毎に異なる可能性

## 6. 温度管理の詳細要件（製品カテゴリ別）

### 6.1 レトルト食品
```
殺菌処理:     121℃ ± 0.5℃ (F値3.0以上)
冷却開始:     殺菌後直ちに開始
冷却終了:     中心温度40℃以下
保管温度:     常温（15〜25℃）
保存期間:     製造日から12〜18ヶ月
```

### 6.2 チルド総菜
```
加熱調理:     中心温度75℃以上、1分以上
冷却開始:     調理完了後30分以内
冷却終了:     中心温度10℃以下（4時間以内）
保管温度:     0〜5℃
保存期間:     製造日から3〜7日（製品別設定）
```

### 6.3 デリカ
```
調理:         加熱しない場合あり（サラダ・マリネ等）
冷却:         必要に応じて迅速冷却
保管温度:     0〜4℃（目標3℃以下）
保存期間:     製造日から1〜3日（製品別設定）
```

### 6.4 おせち料理
```
各構成品のCCPにカテゴリ別に準拠
保管温度:     レトルト品:常温 / チルド品:0〜5℃
保管温度混在: 多層タッパーで区分保管または別倉庫
販売期限:     商習慣に従う（12月31日まで）
```
