# 食品製造業MES - 製造実行要件

## 1. 目的
食品製造の生産計画から実績集計までの全工程を管理する製造実行機能の要件を定義する。

---

## 2. 基本設計方針

### 2.1 バッチ単位生产管理
- 食品製造は「バッチ（ロット）」単位で生産管理を行う
- 各バッチに固有のBatch IDを付与
- BOM（部品表）に基づき原材料を引き当てて生産指示を発行

### 2.2 レシピ・製法書の管理
- 製品ごとにBOMと工程フローを登録
- バージョン管理（変更履歴を追跡）

---

## 3. 機能要件

### 3.1 BOM（部品表）管理

| ID | 要求項目 | 詳細 |
|----|---------|------|
| BM-001 | 製品別BOM登録 | 完成品に必要な原材料・包装資材の構成を定義 |
| BM-002 | バージョン管理 | BOMの変更履歴を追跡（変更日、担当者、理由） |
| BM-003 | 代替原材料の設定 | メイン原料に替代可能な材料を登録 |
| BM-004 | 配合比率 | 各原材料の配合割合（%または重量）を設定 |

### 3.2 工程フロー管理

| ID | 要求項目 | 詳細 |
|----|---------|------|
| PF-001 | 工程定義 | 原料仕入→検収→保管→調合→加熱→充填→密封→冷却→包装→完成品検査→倉庫入库 |
| PF-002 | CCP/OPRPの紐付け | 各工程にCCP/OPRPを割り当て |
| PF-003 | 標準作業時間 | 各工程の標準所要時間を設定（生産計画の基準として） |

### 3.3 生産指示書

| ID | 要求項目 | 詳細 |
|----|---------|------|
| WI-001 | 指示書発行 | 受注/在庫状況からBOMに基づき原材料を引き当て、生産指示を作成 |
| WI-002 | FIFO適用 | 原材料の引き当てはFIFO順 |
| WI-003 | 資材不足警告 | BOMの原材料が在庫不足の場合、発行前に警告 |
| WI-004 | レシピ添付 | 生産指示に標準レシピ・製法書を自動添付 |

### 3.4 作業記録（生産実績）

| ID | 要求項目 | 詳細 |
|----|---------|------|
| WR-001 | 工程毎の開始/終了時刻 | 各工程の实际稼働時間を記録 |
| WR-002 | 原材料使用量の記録 | 実際投入量を記録（計画量との比較も可能） |
| WR-003 | CCP監視値の入力 | HACCPのCCP実測値をバッチ毎に記録 |
| WR-004 | 副産物/廃棄物の記録 | 工程発生量の記録 |

### 3.5 バッチトレーサビリティ

| ID | 要求項目 | 詳細 |
|----|---------|------|
| BT-001 | バッチID発行 | 各生産バッチに固有のIDを自動採番 |
| BT-002 | 原材料ロットとの紐付け | 使用した原材料のLot IDを記録 |
| BT-003 | 製品ロットへの継承 | 完成品のLot IDを作成し、親バッチと関連付ける |

### 3.6 生産実績レポート

| ID | 要求項目 | 詳細 |
|----|---------|------|
| PR-001 | バッチ別実績 | 各バッチの投入量・完成量・歩留まりを表示 |
| PR-002 | OEE計算 | 稼働率・性能効率・良品率の統合指標 |
| PR-003 | 生産計画対実績 | プランとアクチュアルの比較レポート |

## 4. カテゴリ別BOM構造・工程フローテンプレート

本工場で扱う4つのカテゴリ別に、代表的なBOM構造と工程フローを定義する。

---

### 4.1 レトルト食品（代表: レトルトカレー）

| SKU | 商品名 | パッケージ | 賞味期限 | 保管温度 |
|-----|--------|-----------|---------|---------|
| RC-001 | レトルトカレー・スパイスチキン | 250gレトルト袋 | 製造日から18ヶ月 | 常温(15〜25℃) |

**BOM構造:**
```
完成品: RC-001 レトルトカレー・スパイスチキン
├─ 原材料
│  ├─ RM-CHICK01  鶏肉（塊）      30.0 kg
│  ├─ RM-SAUCE01  カレーソース    25.0 kg
│  ├─ RM-VGTMIX01 野菜ミックス    15.0 kg
│  └─ RM-SPICE01  カレースパイス   0.3 kg
├─ パッケージ資材
│  ├─ PK-RET01   レトルト袋(250g) 500 個
│  ├─ LB-CURRY01  ラベル          500 枚
│  └─ BOX-RT01   段ボール（10入）  50 箱
└─ 副資材
   └─ WATER01    調理用水         10.0 kg
```

**工程フロー:**
```
仕入検収 → 原材料保管(常温)
    ↓
前処理（解凍・切り分け）→ CCP-2: 異物混入チェック
    ↓
調合（バッチミキサー）
    ↓
充填（レトルト袋充填機）
    ↓
密封（シーラー）→ CCP-2: 密閉検査
    ↓
殺菌（レトルトオートクレーブ）→ **CCP-1**: 中心温度121℃/F値3.0以上(必須)
    ↓
冷却（水噴射冷却）→ CCP-3: 冷却速度記録
    ↓
乾燥 → パックラベリング
    ↓
箱詰め → 完成品検査 → 常温倉庫入库
```

### 4.2 チルド総菜（代表: 焼肉のたれ煮）

| SKU | 商品名 | パッケージ | 賞味期限 | 保管温度 |
|-----|--------|-----------|---------|---------|
| TC-001 | 焼肉のたれ煮 | 200gメタル容器 | 製造日から7日 | 冷蔵(0〜5℃) |

**BOM構造:**
```
完成品: TC-001 焼肉のたれ煮
├─ 原材料
│  ├─ RM-BEEF01   牛肉（薄切り）   25.0 kg
│  ├─ RM-DAIKON01 大根             8.0 kg
│  ├─ RM-TARE01   たれ（調味液）    10.0 kg
│  └─ RM-OIL01    食用油            2.0 kg
├─ パッケージ資材
│  ├─ PK-MTL01   メタル容器(200g) 500 個
│  ├─ FILM-MTL01 密封フィルム     500 枚
│  └─ LB-TARE01  ラベル           500 枚
└─ 副資材
   └─ WATER01    調理用水          5.0 kg
```

**工程フロー:**
```
仕入検収 → 原材料保管(冷蔵)
    ↓
前処理（解凍・切り分け）→ CCP-2: 温度管理記録
    ↓
炒め工程
    ↓
加熱調理 → **CCP-1**: 中心温度75℃以上/1分以上(必須)
    ↓
充填（メタル容器）
    ↓
密封（フィルムシール）→ CCP-4: シール強度検査
    ↓
冷却 → **CCP-2**: 60℃→10℃まで4時間以内(必須)
    ↓
ラベリング（賞味期限印字）
    ↓
箱詰め → 完成品検査 → 冷蔵倉庫入库
```

### 4.3 デリカ（代表: チキンサラダ）

| SKU | 商品名 | パッケージ | 賞味期限 | 保管温度 |
|-----|--------|-----------|---------|---------|
| DL-001 | チキンサラダ | 150gカップ | 製造日から3日 | 冷蔵(0〜4℃) |

**BOM構造:**
```
完成品: DL-001 チキンサラダ
├─ 原材料
│  ├─ RM-COOKCHICK  茹で鶏肉            20.0 kg
│  ├─ RM-MAYO01     マヨネーズ          5.0 kg
│  ├─ RM-CORN01     コーン              3.0 kg
│  ├─ RM-LETTUCE01  レタス              2.0 kg
│  └─ RM-VEGDRIP    ベジドリーフ        1.0 kg
├─ パッケージ資材
│  ├─ PK-CUP01     カップ(150g)       500 個
│  ├─ FILM-CUP01   密封フィルム       500 枚
│  └─ LB-SALAD01   ラベル             500 枚
└─ 副資材
   └─ ICEPACK01    アイスパック(冷却)   500 個
```

**工程フロー:**
```
仕入検収 → **CCP-1**: 原材料温度・COA確認(必須)
    ↓
原材料保管(冷蔵)
    ↓
洗浄 → CCP-2: 水道水残存塩素濃度チェック(1時間毎)
    ↓
切り分け（スライサー）
    ↓
茹で工程（鶏肉）→ CCP-3: 中心温度確認
    ↓
冷却（急速冷却）
    ↓
混合調理（マヨネーズ等添加・ミキシング）
    ↓
充填 → CCP-4: 充填温度・重量チェック
    ↓
密封 → シール強度検査
    ↓
ラベリング（賞味期限印字）→ CCP-5: アレルギー表示確認(必須)
    ↓
箱詰め + アイスパック同梱 → 冷蔵倉庫入库
```

### 4.4 おせち料理（代表: 重箱1〜3段構成）

| SKU | 商品名 | パッケージ | 販売期限 | 保管温度 |
|-----|--------|-----------|---------|---------|
| OS-001 | おせち重箱(1段目: 黒豆+数の子) | 木製重箱 | 12月31日まで | 常温+冷蔵混在 |

**BOM構造:**
```
完成品: OS-001 おせち重箱・1段目（構成食）
├─ 構成品（完成品として別途生産されたもの）
│  ├─ RC-BLACKBEAN  黒豆レトルト        (SKU: RC-BB01)  ×2袋
│  │   └─ 原材料: コメツク(大豆), サトウ, ショウガ ...
│  └─ DL-KAZUNO     数の子（冷蔵）       (SKU: DL-KZ01)  ×1枚
│      └─ 原材料: タラ卵, 塩, 砂糖 ...
├─ パッケージ資材
│  ├─ BOX-KJ01     木製重箱(1段)        1 個
│  ├─ PAPER01      桐敷紙               1 枚
│  └─ TAG-OS01     商品タグ             1 枚
└─ ギフト包装（任意）
   └─ GIFTBOX01    ギフト用外箱         1 個
```

**工程フロー:**
```
各構成品の個別生産 → カテゴリ別の工程フローに従う
    ↓
完成品検査（各構成品）
    ↓
組み立て工程
    ├─ CCP-X: アレルギー表示確認(必須)
    ├─ CCP-Y: 異物混入チェック（金検）
    └─ CCP-Z: 賞味期限印字の整合性確認
    ↓
重箱詰め（構成順に配置）
    ↓
外装包装 → ギフト包装（オプション）
    ↓
完成品検査 → 倉庫入库(温度区分保管)
```

**おせち特有の製造管理ポイント:**
- **生産集中**: 12月1日〜20日の20日で年間売上の8割以上を処理。人員・設備のスケジューリングが重要
- **構成品の在庫先行確保**: おせち用BOMは前月（11月中）に全構成品の原材料発注完了
- **箱単位Lot ID**: 重箱全体に1つのLot ID、各構成品には個別Lot IDを紐付け
- **賞味期限の商習慣**: 通常は製造日からの日数ではなく「12月31日まで」等の固定日で設定
