## 1. ベース技術スタック（ユーザー指定）

| 層 | 技術 | バージョン目標 | 備考 |
|----|------|---------------|------|
| フロントエンド | Node.js + Vite | Node.js 22 LTS, Vite 7+ | SPA構成を想定 |

## 2. 推奨追加フレームワーク・ライブラリ

### 2.1 フロントエンド

#### UIフレームワーク（決定済み）
- **Vue 3.5+ (Composition API + `<script setup>`)**
- 型安全を重視するため **TypeScript 5.x** と併用

#### コンポーネントライブラリ（決定済み）
- **Element Plus 2.x** — Vue 3ネイティブ、表形式データ管理に最適。日本語ローカライズ対応

#### 状態管理
- **Pinia** — Vue 3公式推奨状態管理ライブラリ、TypeScript対応
- **VueUse** — ユーティリティフック集（useFetch, useLocalStorage等）
- **Vite-plugin-pwa** — PWA化オプション

#### チャート・可視化
- **Apache ECharts via vue-echarts** — センサーデータ時系列表示に最適、リアルタイム更新対応

#### form管理
- **VueUse UseForm（またはvee-validate 4.x）+ Zod** — バリデーション統合

#### HTTPクライアント
- **Axios** — インターセプター機能で認証トークン自動付与（Vueとも互換）

#### リアルタイム通信
- **MQTT.js** — センサーデータをEMQX経由で直接購読（WebSocketブローカー利用）
- または **EventSource (Server-Sent Events)** — バックエンドからのプッシュ通知用

#### PDF生成（出荷伝票等）
- **@vue-office/pdf** または **jsPDF + html2canvas** — ブラウザ内PDF生成

---

### 2.2 バックエンド

#### データベースアクセス（決定済み）
- **jOOQ 3.19+** — 単独で完結。型安全SQL、自動生成DAOによるコンパイル時検証

#### 認証・認可
- **Spring Security + JWT (jjwt)** — トークンベース認証
- **BCrypt** — パスワードハッシュ化

#### バリデーション
- **Jakarta Bean Validation (Hibernate Validator)** — DTO入力バリデーション

#### トランザクション管理
- **Spring @Transactional** — デフォルトで十分、分布式トランザクションは別途検討

#### 設定管理
- **application.yml** + **Spring Profiles** — dev/test/prod環境分け

---

### 2.3 IoT・センサーデータ連携

| 技術 | 用途 | 推奨 |
|------|------|------|
| MQTTブローカー | センサーデータ受信 | **EMQX CE**（決定済み。100台規模対応）|
| MQTTクライアント | jOOQ + Spring | Eclipse Paho MQTT Client for Java |
| Time-Series DB | 履歴保存・クエリ | **PostgreSQL 17 のみ**。JSONB + テーブルパーティションでセンサー履歴を一元管理 |
| キャッシュ/ pub-sub | センサー状態・アラート | **Redis 7+** |
| メッセージキュー | バッチ処理・非同期連携 | **Apache Kafka 3.x**。イベントソーシング基盤として活用

### 2.4 テスト戦略

#### バックエンド（Java）

| テスト種別 | ツール | 目的 |
|-----------|--------|------|
| ユニットテスト | **JUnit 5 + Mockito** | 各クラス単位のロジック検証 |
| インテグレーションテスト | **Spring Boot Test** | レイヤー間の連携検証 |
| DBテスト | **Testcontainers** | リアルPostgreSQLでのE2Eテスト |
| APIモック | **WireMock** | 外部IF（ERP/WMS/LIMS）のモック |
| カバレッジ | **JaCoCo** | コードカバレッジ計測 |

#### フロントエンド（Vue 3 + Vite）

| テスト種別 | ツール | 目的 |
|-----------|--------|------|
| ユニットテスト | **Vitest** | Viteとネイティブ統合、Jest互換 |
| コンポーネントテスト | **@vue/test-utils 2.x + Testing Library (Vue)** | ユーザー行動ベースの検証 |
| E2Eテスト | **Playwright** | ブラウザ自動操作、複数ブラウザ対応 |
| スナップショット | Vitest内蔵 | UI変更検知 |
| カバレッジ | Vitest内蔵 | コードカバレッジ計測 |

## 5. 実装フェーズの推奨順序（決定済みスタック反映）

### Phase 0: 環境構築
- リポジトリ構成（Vue 3 + Vite / Spring Boot + jOOQ / PostgreSQL）
- Docker Compose: PG17, EMQX, Kafka
- GitHub Actions CIパイプライン
- コード規約設定（ESLint/Prettier/SpotBugs）

### Phase 1: 基盤機能
- 認証認可（Spring Security + JWT）
- ファクトリ/ライン管理（3工場初期値）
- ユーザー権限・ロール管理
3. **Phase 2: マスタ管理** — 製品・原材料・サプライヤ・BOM
4. **Phase 3: 在庫・LOT管理** — FIFO・ロット分割・保管場所追跡
